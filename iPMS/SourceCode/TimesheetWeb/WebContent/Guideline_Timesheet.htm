1.    Report Timesheet

1.1             Purpose

This feature is used for all users. In this feature, users can input new, view, update and delete their timesheet.

1.1.1.                    Description

To access this feature, user can:

-          Login with Location is Timesheet.

-         Or, select menu Timesheet

a.     Report Timesheet

This view displays users timesheet with status is Unapproved as default. If he has any rejected timesheet, a message will be displayed to warn him in this view. Also in View Timesheet, user can see records which projects status is On-going or Closed.

If timesheets status is Approved (by PL, or by QA, or by MISC), the record of timesheet is dimmed to prevent from modifying.

If timesheets status is Rejected, the record of timesheet is marked as red.

From here, user can add new, update, or delete his timesheet(s).

b.     Add new Timesheet

User adds the new timesheet(s) everyday. To add a new timesheet, user selects menu item Add New, or click Add New button in View Timesheet.

In Add New Timesheet form, user input information for a record of timesheet. That is:

Project: a project assigned to him, or an internal project in his group, or a project of everyone in FSOFT, or the MISC project.

-          Process: process of work.

-          Type: type of work.

-          Product: a product that work produces.

-          Time: duration spent for that work (from 1 to 8 hours).

-          Description: description of that work (as detail as possible).

After adding, his timesheet status is Unapproved.

c.      Update Timesheet

If users timesheet has something wrong, or timesheets status is Rejected, he can correct it as below:

-          Select incorrect timesheet(s).

-          The Update Timesheet view displays selected timesheet(s) for updating.

After updating, his timesheet status is Unapproved.

d.     Delete Timesheet

User deletes his timesheet(s) by select them, and click Delete button in the Report Timesheet view.

2.    Approve for External Projects

2.1             Purpose

This feature is used for all project leaders and group leaders. After developers input their timesheet, project leader or group leader is responsible for approving or rejecting project members timesheet. And this feature supports for this purpose. In this feature, project leader or group leader can update, approve and reject record(s) of timesheet.

2.2             Description

To access this feature, PL (or GL) can:

-          Login with Location is Approve for Billable Projects.

-          Or, select menu Approve | Approve for Billable Projects.

 

a.     Approved for External Projects

This view displays timesheets of members in his external project. These timesheets have type between 0 and 7. He is responsible for approving the timesheets. If any is incorrect, he can update or reject it.

-          To approve timesheet, he selects it first, and then click Approve button in Approved for External Projects view. After approving, timesheets status is ApprovedByPL.

-          If timesheet is incorrect, he can reject and enter the commend for this timesheet. After rejecting, timesheets status is Rejected.

b.     Update Timesheet by Project Leader.

To update timesheet(s):

-          Select timesheet(s) in Approved for External Projects view, the selected total is not higher than 25 to avoid overloading system resource when displaying in Update Timesheet by PL form.

-          He updates in the Update Timesheet by PL form.

After updating, timesheets status is ApprovedByPL.

3       Approve by Quality Assurance

3.1             Purpose

This feature is used for all QAs. After Project Leader (or Group Leader) approves timesheet of each member in his team, QA is responsible for updating and approving their timesheets. And this feature supports for this purpose.

3.2             Description

To access this feature, QA can:

-          Login with Location is Approve by Quality Assurance.

-          Or, select menu Approve | Approve by Quality Assurance.

a.     Approve Timesheet by Quality Assurance

This view displays timesheets of all external projects that were approved by PLs (or GLs). Status of the timesheets is ApprovedByPL.

-          If timesheet had KPA, QA can approve it.

-          To approve, he selects timesheet(s). Then, click Approve button.

-          After approving, timesheets status is ApprovedByQA.

-          To reject timesheet, he selects it first. Click Reject button.

After rejected, timesheets status is Rejected.

b.     Update Timesheet by Quality Assurance

To update timesheet(s):

-          Select timesheet(s) in Approved Timesheet by Quality Assurance view, the selected total is not higher than 25 to avoid overloading system resource when displaying in Update Timesheet by QA form.

-          He updates in the Update Timesheet by PL form with KPA value(s) for timesheet(s).

-          After updating only, timesheets status is the same as before.

-          After updating and approving, timesheets status is ApprovedByQA.

4       Approve for Internal Projects

4.1             Purpose

This feature is used for Group Leaders or Project Leaders who lead internal projects. This feature is used for approving all timesheets that have project type is non-billable project.

4.2             Description

To access this feature, GL (or PL) can:

-          Login with Location is Approve for Internal Projects.

-          Or, select menu Approve | Approve for Internal Projects.

a.     Approve for Internal Projects.

This view displays timesheets of internal projects in GLs group and MISC project, or internal projects PL leads. These timesheets have type 8, or 9. This view does not display MISC or internal projects of other groups.

GL (or PL) is responsible for approving the timesheets. If any is incorrect, he can update or reject it.

-          To approve timesheet, he selects it first, and then click Approve button in Approved for Internal Projects view. After approving, timesheets status is ApprovedByMISC.

-          If timesheet is incorrect, he can reject. After rejecting, timesheets status is Rejected.

b.     Update Timesheet by Other

To update timesheet(s):

-          Select timesheet(s) in Approved for Internal Projects view, the selected total is not higher than 25 to avoid overloading system resource when displaying in Update Timesheet by Other form.

-          He updates in the Update Timesheet by Other form.

After updating, timesheets status is ApprovedByMISC.

5       Reports

5.1             Purpose

This feature is used for all developers to collect metric into projects database, or monitor efforts. And this feature is read-only.

5.2             Description

To access this feature, user can:

-          Select one menu item on menu Report.

There are 3 types of report: Summary, Inquiry, and Weekly.

a.     Summary Report

To collect metric into projects database, user uses Summary Report. In Summary, there are 5 tables which displays total efforts depend on the selected project and timesheets status. Using searching parameters, he can filter out his needed values.

-          Summary table displays Total, Correction, Management, Quality, and Development efforts. In addition there is a Total row whose value is equal to following formula.

Total = Effort (Development + Quality + Management)

-          Process-TypeOfWork table displays total efforts whose Process and TypeOfWork are matched. In addition there are Total row and column to sum of corresponding value.

-          WorkProduct-TypeOfWork table displays total efforts whose WorkProduct and TypeOfWork are matched. In addition there are Total row and column to sum of corresponding value.

-          KPA-TypeOfWork table displays total efforts whose KPA and TypeOfWork are matched. In addition there are Total row and column to sum of corresponding value.

-         Account-Date table displays efforts in selected period of developers in the group. In addition there are Total column to sum all effort of each developer in the period.

-         Project-Process table displays efforts in selected period of projects in the group distributed by process. In addition there are Total column to sum all effort of each project in the period.

-   Project-WorkProduct table displays efforts in selected period of projects in the group distributed by work product. In addition there are Total column to sum all effort of each project in the period.

-   Project-KPA table displays efforts in selected period of projects in the group distributed by KPA. In addition there are Total column to sum all effort of each project in the period.

-   Project-TypeOfWork table displays efforts in selected period of projects in the group distributed by type of work. In addition there are Total column to sum all effort of each project in the period.

b.     Inquiry Report:

User can view all timesheets and status. And he can update them.

c.      Weekly Report:

There are 2 tables (Lack of Timesheet, and Pending Timesheet) to display user(s) who lack of timesheet, or timesheet(s) are pending.

-          Lack Of Timesheet View displays a list of developers in the group who are lack at least 1 timesheet in selected period. Lack dates are marked the sign X.

-          Pending Timesheet View displays developers in projects whose timesheets are pending (not be approved).
