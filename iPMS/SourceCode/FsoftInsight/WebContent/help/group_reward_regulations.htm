Operation groups

The purpose of this regulation is to motivate and evaluate operation groups periodically.

Source of information

Group Plans and Group Report
Project points of post mortem projects in reported period
Support Groups

Group Evaluation

Max is 300 points, Min is 0 point
Group Point = Production Point + Business Point + Staff Point

Production Points

Max is 100 points, Min is -50 point
Production Point = Project Point + Proposal Point
Project Point = Average of Project Points *1.4 if PWR>0, = Average of Project Points if PWR >-30, =-30 if PWR <-30
Proposal Point = (Proposal Win Rate (%)-20)*k, where k=0.375 if PWR>20, =1 if PWR<20

Business Points

Max is 100 points, Min is -50 point
Business Point = Revenue Points + Profit Points + Receivable Points
Revenue Point = Revenue Performance (%) *0.2 if RP<175%, =35 if RP>175, =-10 if RP<30
Net Income Point = Min(50,Net Income Ratio (%)*k, where k = 0.5 if NIR>10, = 0 if  NiR>0, =-10 if NIR<0)
Receivable Point = Min(15,(Receivable Ratio (%) -70)*k), where k = 0.5 if RR>70, =0.2 if RR >50, = 0.5 if RR>30, = 1 if  RR<30

Staff Points

Max is 100 points, Min is -50 point
Staff Point = Language Point + Experience Point - Turnover Point + Technology Point
Language Point = (Language Index-2) *5, =15 if LI>5
Experience Point = (Experience Index -1)*5 if EI<10, =45 if EI>10
Turnover Point = (Voluntary Turnover Rate (%)- 10) *2 if VTR<25, =-30 if VTR>25
Technology Point = (Technology Index-1) *5 if TI<5, =20 if TI>5

Summary

Perfect Fighters (Quarterly): Highest production point
Gold Miners (Quarterly): Highest business point
Excellent Team (Quarterly): Highest staff point

BA groups

The purpose of this regulation is to motivate and evaluate Fsoft support groups periodically.

Group Evaluation

Max is 300 points, Min is 0 point
Group Point = Production Point + Business Point + Staff Point

Production Points

Max is 100 points, Min is -50 point
Production Point = Action Point + Devotion Point
Action Point =25+(Timeliness-LCL of norm of the Timeliness)/(UCL-LCL)*50 if LCL<=Timeliness<=UCL, = 0 if Timeliness<LCL, = 75 if Timeliness >UCL, = -35 if the metric is not calculated
Devotion Point =5+(UCL of norm of the Response time - Response time)/(UCL-LCL)*20 if LCL<=Response time<=UCL, = 0 if Response time>UCL, = 25 if Response time<LCL, = -15 if the metric is not calculated

Business Points

Max is 100 points, Min is -50 point
Business Point = Saving Point + Busy Point + Customer Point + Value Point
Saving Point = 5+(UCL of norm of the Budget performance - Budget performance)/(UCL-LCL)*10 if LCL<=Budget performance<=UCL, = 0 if Budget performance>UCL, = 15 if Budget performance<LCL, = -5 if the metric is not calculated
Busy Point =5+(Busy rate-LCL of norm of the Busy rate)/(UCL-LCL)*10 if LCL<=Busy rate<=UCL, = 0 if Busy rate<LCL, = 15 if Busy rate >UCL, = -5 if the metric is not calculated
Customer Point =20+(Customer satisfaction-LCL of norm of the Customer satisfaction)/(UCL-LCL)*25 if LCL<= Customer satisfaction<=UCL, = 0 if Customer satisfaction<LCL, = 45 if Customer satisfaction >UCL, = -15 if the metric is not calculated
Value Point =25+(Achivement-LCL of norm of the Achivement)/(UCL-LCL)*50 if LCL<= Achivement<=UCL, = 0 if Achivement<LCL, = 75 if Achivement >UCL, = -25 if the metric is not calculated

Staff Points

Max is 100 points, Min is -50 point
Staff Point = Language Point + Technology Point
Language Point =5+(Language index-LCL of norm of the Language index)/(UCL-LCL)*20 if LCL<= Language index<=UCL, = 0 if Language index<LCL, = 25 if Language index >UCL, = -25 if the metric is not calculated
Technology Point =5+(Technology index-LCL of norm of the Technology index)/(UCL-LCL)*20 if LCL<= Technology index<=UCL, = 0 if Technology index<LCL, = 25 if Technology index >UCL, = -25 if the metric is not calculated

Summary

Excellent: >225 points
Good: >175 points
Fair: >125 points
Acceptable: >75 points
Failed: <=75 points

 

Project

The purpose of this regulation is to motivate project teams periodically
Any Metric or Record, which is not be available in project, should be defined by permitted tailorings/deviations

Project Evaluation

Project Point = Efficiency Point + Quality Point + Record Point + Special Point
Efficiency = Max 15 points, Min -5 points
Efficiency Point = Size Point (in UCP) + Effort Point
Size Point = Max (Min(10,(Project Size - Norm Size)* 0.1),0)
Effort Point = -5, if project has not recorded accurately the Effort
If Effort Effectiveness (%) >= 110, Effort Point = 0
If Effort Effectiveness (%) >= 100 and <110, Effort Point = 2
If Effort Effectiveness (%) < 100, Effort Point = 5

Quality

Max 15 points, Min -15 point
Quality Point = Sum of Points of Metric (Customer Satisfaction, Acceptance of Deliverables, Requirement Completeness, Timeliness, Leakage, Correction Cost, Process Compliance)
Metric Point = -2, if project has not recorded accurately the Metric
If Metric is better Control Limits of Norm, Metric Point = 3 for Customer Satisfaction and = 2 for others
If Metric in Control Limit Rank of Norm, Metric Point = 1
If Metric is worse Control Limits of Norm, Metric Point = -3 for Customer Satisfaction and = -2 for others

Record

Max 10 points, Min -10 points
Record Point = Sum of Doc Points (Work Order, Acceptance Note, Project Plan, Project Reports)
If project has required Document in time, Doc Point = 2 for Work Order and Acceptance Notes and = 1 for others
If project has no required Document, Doc Point = -2 for Work Order and Acceptance Notes and = -1 for others
If project has delayed the approval of Work Order or Acceptance Note or Project Plan, Doc Point = -1
If project has delayed each Report (Milestone, Post Mortem), Doc Point = -0.5

Special

Max 10 points, Min -10 points
Special Point = Sum of Customer Points + Prestige Point
If project has no special customer requirements on quality, Customer Point = 0
If project achieves a special customer requirement on quality, Customer Point = 2
If project does not met a special customer requirement on quality, Customer Point = -1
VP Operation give the Prestige Point based on contribution of project to Fsoft prestige or value

 Project Reward

A project is Failed if:
- Acceptance of Deliverables < 50%, or
- Effort Effectiveness > 120%, or
- Project Point <= 0
Otherwise a project is Success

Success Level of the Project depend of project points:
Excellent           >30 point
Good   <=30 and >20 points
Fair      <=20 and >10 points
Acceptable       <=10 and >0 points
Default value = 12 points
