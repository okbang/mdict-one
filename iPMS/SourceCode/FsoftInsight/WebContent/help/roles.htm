Roles

Roles are entities that define the access rights for sections of the application.
Roles names can be for example : administrator, group leader, programmer, project leader, tester ...
Roles only apply after they are associated with a Work unit in the User profile section.

Creating a role

The easiest way to understand the role concept is to take an example, let's say that the customer of project x wants to have access to the project x data, we need to create a role called "customer".

First we will login the application using an account with administrator rights, then we will click on the Admin tab, and click on  Roles icon in the left menu. A page will appear with the current list of Roles defined. On the bottom of the list, please click on Add New button. A new page will appear where we will define the new Role :

- Role name: Input here the chosen role name, in our example "Customer". Note: this name must be unique in the role list

- Description : Input here some short description of your choice

- Dashboard Role: This field is used by Dashboard application, if Dashboard is required, please  refer to Dashboard documentation, otherwise, the value doesn't matter.

- Below the previous fields, there is a list of items divided in 2 columns, in the right columns are listed the application sections' names  (or Pages), the left column is the right that will be granted to the Customer on these sections. There are 3 types of rights :

    * No right (column 1) means that the Customer will not be able to view the contents of the section

    * View only right (column 2) means that the Customer will be able to view the contents of the section, but will not be allowed to change the data of this section.

    * Manager right (column 3) means that the Customer will be able to view the contents of the section and allowed to change the data of this section.

In our example, the Customer doesn't need to change any data, and just needs to view the data of his project X. So, we will grant the Customer role  the View only right on all project related sections (all sub-sections under the section Project home), we will check No right on all other sections.

- Click Ok button in order to validate the Customer Role creation.

- If we want to finish the example, the next steps are : Create User profile for the Customer and then grant him rights .
