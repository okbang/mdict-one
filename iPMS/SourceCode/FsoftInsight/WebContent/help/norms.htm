Norms

Norms are the organization standard values for various metrics.
The metrics meaning can be found at :
file://///fsoft-filesrv/Fsoft-QMS/Document/Fsoft/Software/Guideline/Guideline_Software metrics .doc

Norms are made of three values:
LCL: Lower Control Limit.
UCL: Upper Control Limit
Norm: The value of the norm

A metric is considered in-control when it's actual value is between LCL and UCL. When the value of the metric is out of these boundaries, it should be paid special attention to analyze and eventually take action to bring the value back  in-control.
The Norm value is the target between LCL and UCL.

Distribution metrics: These special norms don't have LCL and UCL.

Norms are defined for a period of six month. Projects will be assigned a set of norms depending on which set of norm is defined for their lifecycle at the time of their Actual start date.

When a norm is not yet set, the suggested value (from the previous period) will appear in red. In order to apply these values, please save the norms
